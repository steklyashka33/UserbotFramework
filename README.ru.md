**Масштабируемый фреймворк для распределенных Telegram юзерботов с умным управлением сессиями.**

---

**CoreUserbot** — это готовое к продакшну модульное решение для создания распределенных Telegram юзерботов. Проект предоставляет надежный фундамент для управления множеством аккаунтов с изоляцией процессов, высокой отказоустойчивостью и защитой от банов.

### ✨ Ключевые Особенности
*   ⚙️ **Распределенная Архитектура:** Независимые сервера Менеджера (FastAPI) и Бота (Aiogram).
*   🛡️ **Умный Жизненный Цикл:** Класс `Account` (ООП) с проактивными проверками здоровья сессий.
*   🧠 **"Человеческое" Поведение:** Встроенная процедура "прогрева" (warm-up) для имитации активности реального приложения.
*   🎭 **Защита от Банов:** Движок обфускации сообщений на базе Unicode для обхода автоматических систем обнаружения.
*   🌐 **Multi-Host Ready:** Централизованная настройка сети с поддержкой работы Бота и Менеджера на разных серверах.
*   🚀 **Мгновенный Запуск:** Запуск портов API без ожидания авторизации сессий (в фоне через Zero-Wait Indexing).
*   🐋 **Docker-ready:** Готовность к деплою одной командой.

### 📦 Установка

#### Вариант 1: Docker (Рекомендуется)
1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-username/CoreUserbot.git
   cd CoreUserbot
   ```
2. **Настройте переменные окружения:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env, указав API_ID, API_HASH и Bot Token
   ```
3. **Запустите:**
   ```bash
   docker-compose up -d --build
   ```

#### Вариант 2: Локальный запуск
1. **Подготовьте окружение:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # На Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```
2. **Конфигурация:**
   Скопируйте `.env.example` в `.env` и заполните его. При необходимости настройте сетевые адреса в `shared/config.py`.
3. **Запуск:**
   ```bash
   python main.py
   ```

### 📂 Структура Проекта
```text
.
├── bot_server/          # Интерфейс управления на базе Aiogram
│   ├── obfuscator.py    # Движок защиты текста (Unicode)
│   └── handlers.py      # Единая логика взаимодействия с API

│   └── core/            # Инкапсулированная логика Account (Telethon)
├── shared/              # Общий конфиг, логгер и утилиты
├── sessions/            # Хранилище .session файлов
├── main.py              # Точка входа и оркестратор процессов
└── docker-compose.yaml  # Конфигурация деплоя
```

### ⚙️ Конфигурация
Проект использует гибридную систему настроек для максимальной гибкости:

1.  **.env**: Хранение секретных ключей (`API_ID`, `API_HASH`, `API_TOKEN`).
2.  **shared/config.py**: Единый источник правды для всех остальных параметров:
    *   `MANAGER_HOST` / `BOT_HOST`: IP-адреса для работы на разных серверах.
    *   `MANAGER_PORT` / `BOT_PORT`: Порты взаимодействия.
    *   `USE_UNIQUE_DEVICES`: Переключение между уникальными (но стабильными) и статичными отпечатками устройств.
    *   `STATIC_DEVICE_CONFIG`: Фиксированный профиль устройства, используемый при отключенной рандомизации.
    *   `HIDE_API_LOGS`: Режим тишины в консоли.
